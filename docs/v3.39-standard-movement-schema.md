# v3.39 표준 동작열 스키마 설계 (1단계)

## 목적
영상 분석의 기준점이 되는 품새 동작열을, 공식 국기원 도해 구조에 맞춰 표준화한다.
사용자 정의 구조: **준비 → 품새시작(1번) → … → 품새끝(N번) → 끝맺음(차렷/준비복귀)**.
준비·끝맺음은 동작 번호에 포함되지 않으나 영상에는 존재하므로 분석 단위에는 포함한다.

## 핵심 원칙
1. **번호동작 수 = 공식 N과 정확히 일치**해야 한다(준비·기합·끝맺음·메모 제외하고 N개).
2. **복합동작**(예: 14-1 앞차기 + 14-2 지르기)은 한 동작(번호 1개)으로 묶되,
   내부 `pulses`(세부 펄스 수)를 기록한다 → 영상 분할 시 펄스 N개를 한 동작으로 묶고,
   공식 감점("복합동작 과정 미충족 = 빠진 과정 수 × −0.3")도 판정 가능.
3. **준비(prep)**: moveNo=0, 분석 단위 포함, 번호 미부여.
4. **끝맺음(finish)**: 마지막 동작 후 제자리/차렷. moveNo=N+1(표시상), 분석 단위 포함.
5. **기합(kihap)**: 번호 미부여. 특정 동작 번호에 귀속되는 '마커'로 기록(kihapAt: 동작번호).
   공식 규칙: 기합 누락·오위치는 감점 대상이므로 위치 검증용.

## 분석 단위(영상에서 끊어야 할 구간) 개수
= 1(준비) + N(번호동작) + 1(끝맺음) = **N+2**
단, 영상에 준비/끝맺음이 없으면 자동 감지로 가감(영상마다 다름 → align 방식).

## 데이터 스키마(품새별)
```
{
  name: "태극 1장",
  officialCount: 18,          // 공식 번호동작 수 N
  kihapAt: [18],              // 기합이 들어가는 동작 번호(복수 가능)
  prep: { name, stance },     // 준비자세
  finish: { name },           // 끝맺음
  moves: [                    // 길이 = officialCount = N
    { no: 1, name, type, pulses: 1, dir },
    ...
    { no: 14, name: "앞차기 후 몸통지르기", type:"kick_punch", pulses: 2 }, // 복합
    ...
  ]
}
```

## 영상 분할 정렬(2단계 예고)
- 영상의 모든 동작 펄스를 탐지(개수 강제 X).
- 준비자세 자동감지: 영상 시작부 긴 정지 → 첫 큰 펄스 직전까지를 prep 구간.
- 끝맺음 자동감지: 마지막 큰 펄스 이후 정지 구간을 finish.
- 가운데 펄스열을 moves에 정렬. 복합동작(pulses≥2) 자리는 펄스를 합쳐 1동작에 매핑.
- 총 펄스 기대치 = Σ moves.pulses (복합 반영). 이 값으로 정렬 신뢰도 판정.

## 신뢰성 지표(3단계 예고)
- 메트로놈: 동작 간 간격의 일관성(표준편차/평균). 국기원 영상 대비 박자 편차.
- 속도: 동작별 peak 속도(몸통길이/초).
- 맺음: 정점 후 감속률(스냅).
- 기합 타이밍: 검출된 음성/큰 동작과 kihapAt 동작번호 일치.

## 1단계 완료 검증 결과 (태극 1장)

구현 완료 항목:
1. 표준 스키마 `standard` 필드를 태극1장에 추가(준비/18동작/끝맺음/복합 pulses/기합 마커).
2. `buildGenericMovements`가 표준 스키마 우선 사용 → 준비(moveNo0)+동작1~18+끝맺음.
   번호동작이 정확히 18개로 공식 N과 일치(기존엔 16개로 어긋났음).
3. `detectMotionBoundaries`를 펄스 인식으로 개선: 복합동작 반영해 Σpulses(=22)개
   펄스 구간을 만든 뒤 `foldPulsesToMoves`로 동작 20개(준비+18+끝맺음)로 접음.
4. `detectActiveMotionRange`에 데이터 공백 가드 추가: 2.5초↑ 무활동 간극으로 시연을
   분리하고 가장 충실한 덩어리만 분석 → 비정상 장구간 제거.

실측 검증(태극1장 캐시, 시연 4.8~23.1초):
- 펄스 22개 탐지 → 동작 20구간으로 정확히 접힘(기대 20).
- 공백 가드가 영상의 8개 덩어리 중 실제 시연만 선택.
- 구간 길이 0.18~2.51초(평균 0.91), 비정상 장구간 1개 이하로 정상화.
  (가드 적용 전엔 24~36초 공백 때문에 15.67초 비정상 구간 발생했음)

→ 표준 동작열 구조가 옳음을 실증. 동일 구조로 나머지 16품새 확장 가능.
