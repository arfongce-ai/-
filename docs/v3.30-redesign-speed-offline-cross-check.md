# v3.30 도장 리디자인 · 속도 · 오프라인 명조 교차검증

## 변경 사항

- 전체 동작 전환점 스캔을 무작위 seek에서 순방향 재생 디코딩(`forwardScanFrames`)으로 변경했습니다.
- 순방향 스캔이 채택하지 못한 목표 시각만 기존 정밀 seek 경로(`waitForSeek`)로 자동 폴백합니다.
- 전통 도장(道場) 무드 리디자인을 적용했습니다(명조 제목, 단청 청·홍, 한지 질감, 다크/라이트 토글).
- 수련품새 리포트를 쉬운 말 피드백으로, 경기품새 리포트를 수정 우선순위 안내로 바꿨습니다.
- 명조 서체(Noto Serif KR)를 글자 서브셋해 로컬에 포함하고 Google Fonts 의존성을 제거했습니다.

## 정확성·신뢰성 보호

- 구간 경계, 활성 구간 탐지, 동작명 매핑, 점수 환산 로직은 변경하지 않았습니다.
- 순방향 스캔의 프레임 채택 허용오차(`FORWARD_SCAN_ACCEPT_SECONDS`)를 기존 seek 허용오차(`SEEK_EPSILON_SECONDS`)와 동일하게 두어, 채택되는 프레임의 시각·좌표 정밀도를 기존 경로와 일치시켰습니다.
- 순방향 스캔으로 채택하지 못한 목표는 기존 정밀 seek로 폴백하므로 결과 프레임의 시각·좌표·개수가 기존과 같습니다.
- 폴백 추론은 순방향 스캔이 사용한 타임스탬프 구간을 넘어선 별도 베이스(`fallbackBaseMs`)를 써서 MediaPipe 타임스탬프 단조 증가 요건을 항상 만족시킵니다.
- `requestVideoFrameCallback` 미지원 환경에서는 순방향 스캔이 즉시 종료되어 전 구간이 기존 seek 경로로 동작합니다.

## 오프라인 동작 보호

- 명조 서체는 현대 한글 음절 전체와 사용 라틴·숫자·문장부호·한자(品勢今日修練道場太極)를 포함하도록 서브셋했습니다.
- SemiBold(500·600), Bold(700·800), Black(900) 세 굵기를 `www/assets/fonts`에 woff2로 포함했습니다.
- HTML에서 `fonts.googleapis.com`·`fonts.gstatic.com` 참조를 제거하고 로컬 `@font-face`로 대체했습니다.
- 서비스워커(`www`·최상단)의 사전 캐시 목록에 세 폰트를 추가하고 캐시 버전을 올렸습니다.
- 외부 네트워크 차단 상태에서 세 폰트가 모두 로드되고 제목이 명조로 렌더링되는 것을 확인했습니다.

## 자동 검증 통과 항목

- `npm run verify:cross` — 신규 항목(로컬 명조 연결·파일 존재·외부 폰트 제거·서비스워커 폰트 캐시·테마 토글·순방향 스캔·쉬운/수정중심 카피)을 포함해 통과합니다.
- `npm run assets:check` — 로컬 명조 3종을 포함한 오프라인 자산 점검을 통과합니다.
- `npm run test:engine` — 합성 분석 엔진 검증을 통과합니다.
- `npm run test:mapping` — 동작-구간 매핑 384회 시뮬레이션을 통과합니다.
- `npm run test:performance` — 프레임 재사용·정밀 폴백·신뢰성 가드 검증을 통과합니다(공통 영상 기준 약 31.3% 포즈 검출 감소).

## 런타임 점검

- 테마 토글, 훈련 목적(수련/경기) 전환, 품새 분류(유급/유품/유단) 전환을 수행하는 동안 콘솔 오류·경고가 발생하지 않음을 확인했습니다.
