# v3.36 7개 샘플 교차검증 · HEVC 감지 교차검증

## 요청

- 7개 샘플로 영상 분석 신뢰성 교차검증.
- 업데이트 후 체감상 변화를 크게 못 느낌.

## 교차검증 방법

업로드된 7개 샘플을 H.264로 변환한 뒤, 앱과 동일한 포즈 모델(`pose_landmarker_lite.task`)·동일 스캔 밀도(3.5/초)·동일 motion 계산으로 프레임을 추출하고, 앱의 실제 하이브리드 함수(`detectActiveMotionRange` / `detectNaturalBoundaries` / `allocateMovementsToSegments`)를 index.html에서 추출해 적용했다.

- 7개 모두 관절 감지율 98~100%로 촬영 품질이 우수하다.
- 활성구간이 합리적으로 잡히고, 동작이 빠짐없이 배분되며, 경계가 단조 증가한다.
- 민감도 0.5~1.0 변화에도 구간 수가 일관(샘플1: 16구간/2묶음 고정)되어 안정적이다.
- 동작 수 가정(18/23/27/30)을 바꿔도 배분이 정확하다.

→ **알고리즘 자체의 신뢰성은 확인됨.**

## 체감 없음의 유력 원인 — HEVC 코덱

업로드된 7개 샘플은 **전부 HEVC(H.265)** 코덱이다(1920×1080 6개, 720×1280 세로 1개). 많은 모바일 브라우저·WebView는 HEVC 디코딩을 지원하지 않는다. 이 경우 영상이 화면에 보여도 분석 단계에서 프레임을 읽지 못해 분석이 진행되지 않거나 빈 결과가 나온다. 그러면 어떤 알고리즘 개선도 체감될 수 없다.

## 수정 — HEVC 감지·경고

- 영상 로드 시 파일의 앞 128KB와 뒤 256KB를 읽어 HEVC 박스(hvc1/hev1/hvcC)를 검사한다. moov(코덱 정보) 박스가 파일 끝에 있는 경우(faststart 미적용)까지 잡기 위해 뒤쪽도 검사한다.
- HEVC이면서 현재 브라우저가 HEVC를 디코딩하지 못하면(`canPlayType` 음성), "H.264(MP4)로 변환 후 재시도"를 명확히 안내한다.
- 영상 디코딩 실패(`error` 이벤트)도 직접 잡아 코덱 문제 가능성을 안내한다.

### 감지 검증

- 실제 업로드 원본 7개: **7/7 HEVC 감지**.
- H.264로 변환한 7개: **오탐 0/7**.

## 자동 검증

- `node scripts/test-hevc-detection.js` — 앞부분 hvc1, 파일 끝 hvcC, H.264 오탐 없음, hev1 변형 감지를 검증한다.
- `npm run verify:cross` — HEVC 감지 함수·끝부분 검사·안내 문구를 검증하며 통과한다.
- `assets:check`, `test:engine`, `test:mapping`(384회), `test:performance`, `test:timestamp`, `test:boundary` 모두 통과한다.
- 페이지 로드·모드 전환에서 콘솔 오류 없음.

## 사용자 권장

1. 분석이 시작조차 안 되거나 결과가 비면, 영상이 HEVC일 가능성이 높다. **H.264(MP4)로 변환**해 다시 올린다.
   - 아이폰: 설정 → 카메라 → 포맷 → "높은 호환성"으로 촬영하면 H.264로 저장된다.
   - 변환: 무료 변환 앱 또는 PC에서 H.264 MP4로 변환.
2. H.264 영상으로 다시 분석하면, 이번 업데이트(하이브리드 경계·동작 묶음·모드별 리포트)의 변화가 체감된다.
3. 촬영은 전신이 화면에 들어오게, 각 동작 끝에서 잠깐 멈춰서 하면 가장 정확하다.
