## 3. v3.38 동작 분할 개선 → 효과 확인, 단 한계 존재

### 3-1. 효과 (검증됨)
정점(peak) 기반 분할로 17개 전 품새의 시연 구간에서 **생성 구간 수 = 공식 동작 수**
달성. 기존 valley 방식은 동일 영상에서 13~19구간으로 부족(과합쳐짐)했음.
경계 시각은 균등분할이 아닌 실제 정지점을 추종(균등 대비 평균 0.8~3.4초 이탈).

### 3-2. ★ 구조적 한계 1 — 복합동작(다과정 동작) 경계 정확도
공식 채점은 "복합동작의 과정 미충족 = 빠트린 동작 수 × −0.3점"으로, 한 동작 번호
안의 여러 과정을 인정함(p_2026.pdf 30면). 코드 동작명도 이를 한 항목으로 묶음
(예: "오른발 앞차고 얼굴반대지르기").

그런데 이런 복합동작은 영상에서 **motion peak가 2개 이상** 발생함(차기 봉우리 +
지르기 봉우리). 17개 품새에 복합동작이 **약 72개** 존재.

peak 분할은 동작 수만큼 봉우리를 강제로 N개 선택하므로 **개수는 맞지만**, 복합동작
구간에서 봉우리 2개 중 1개만 경계로 잡혀 **경계가 동작 중간에 찍힐 수 있음**.
→ "개수 일치 ≠ 경계 위치 정확". 일여(복합 10개) 활성구간에서 peak 후보 159개 중
   23개 선택을 확인. 현 단계에서 경계 위치 오차는 정밀 측정 전임(추후 표본 확대 시 보정).

### 3-3. ★ 구조적 한계 2 — 긴 영상 / 데이터 공백 구간
- 해설·반복이 섞인 긴 교육영상을 통째로 넣으면 활성구간에 비시연 구간이 포함됨.
- pose 미검출 공백(카메라 전환, 화면 이탈)이 있으면 그 자리에 비정상적으로 긴
  구간이 생김. 일여 테스트에서 25~35초 공백으로 15.75초짜리 비정상 구간 발생 확인.
- 현 `detectActiveMotionRange`는 "양끝 저활동만 제거"라 중간 공백·해설은 못 거름.

→ 실사용이 '짧은 1회 시연 영상'이면 문제 없음. 긴 영상은 시연 부분을 잘라 입력 권장.
   (사용자 확인: 참고용 긴 영상은 앱에 넣지 않음 → 실사용 영향 제한적)
