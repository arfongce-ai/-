# v3.33 동작 묶음 표시 · 경계 정확도 교차검증

## 보고된 문제 (실제 영상 기준)

태극 2장 영상에서:
- 4번 오른 앞서기 아래막기 구간에 지르기까지 들어감.
- 5번 구간에 6·7번 동작이 한꺼번에 적용됨.
- 그 뒤로 동작명이 계속 한 칸씩 밀리며 꼬임.

## 원인 (실제 영상 분석으로 확인)

업로드된 태극 2장 영상(30.5초, 18동작)의 포즈를 앱과 동일한 방식으로 추출해 움직임 곡선을 분석한 결과:

- 태극 2장의 "막기→지르기"는 **사이에 뚜렷한 멈춤이 없는 연속 동작**이라, 움직임 속도만으로는 두 동작을 가를 수 없다.
- 준비자세(시선 처리)가 약 3초로 길어 첫 구간이 비대해진다.
- 그런데도 경계를 **동작 수(18개)만큼 억지로 만들면**, 멈춤이 없는 곳에 가짜 경계가 생기거나 한 구간에 두 동작이 묶이면서 이름표가 한 칸씩 밀린다.

## 수정 — 동작 묶음 표시

경계를 18개로 강제하지 않고, **멈춤이 뚜렷한 만큼만 자연스럽게 끊는다**(예: 16구간). 그리고 18개 동작을 **각 구간 길이에 비례해 배분**한다.

- 긴 구간(연속 동작)에는 여러 동작이 배분되어 `A + B`로 묶여 표시되고, 몇 번~몇 번 동작인지 함께 보여준다.
- 짧은 구간은 1동작씩 그대로 표시된다.
- 여러 동작이 묶인 구간이 있으면 리포트 상단에 "N개 동작을 M개 구간으로 묶었다"고 안내하고, 각 동작 끝에서 잠깐 멈춰 촬영하면 분리된다고 알려준다.

## 실제 영상 검증 결과

업로드 영상에 새 로직을 적용한 결과(민감도 0.95):
- 16개 자연 구간으로 끊기고 18개 동작이 길이비례로 배분됨.
- 연속 동작 구간 2곳이 자동으로 묶여 표시됨(예: "11~12번 묶음", "16~17번 묶음").
- 각 구간이 자신이 포함하는 동작을 정직하게 표시 → 이름표가 한 칸씩 밀려 꼬이는 문제 해소.

## 정확도·안전성

- 동작 배분은 합이 정확히 동작 수가 되도록 최대잔여법으로 정수화하고, 각 구간 최소 1동작·순서 보존을 보장한다.
- 멈춤이 거의 없는 영상에서도 최소 1구간 이상을 생성하고 전체 동작을 빠짐없이 배분한다(실패하지 않음).
- 점수 계산은 각 구간의 대표(첫) 동작 기준으로 수행한다.

## 자동 검증

- `node scripts/test-boundary-detection.js` — 자연 구간 수 범위·단조 증가, 동작 배분의 합·순서·최소1개, 길이비례 배분, 멈춤 부족 시 안전 배분을 검증한다.
- `npm run verify:cross` — 자연 경계 탐지, 길이비례 배분, 묶음 수 기록, 묶음 번호 표시, 묶음 안내를 검증하며 통과한다.
- `assets:check`, `test:engine`, `test:mapping`(384회), `test:performance`, `test:timestamp` 모두 통과한다.
- 페이지 로드·모드/품새 전환 중 콘솔 오류 없음.

## 참고: 영상 코덱

업로드 영상은 HEVC(H.265)였다. 일부 모바일 브라우저·WebView는 HEVC 디코딩을 지원하지 않아 분석이 시작되지 않을 수 있다. 분석이 전혀 진행되지 않으면 영상을 H.264(MP4)로 변환해 다시 시도하는 것이 좋다.

## 가장 정확한 결과를 위한 촬영 팁

- 각 동작 끝에서 잠깐(0.3~0.5초) 분명히 멈추면 동작이 하나씩 분리된다.
- 전신이 항상 화면에 들어오도록 촬영한다.
- 동작이 여전히 묶이면 구간 감지 민감도를 높여 더 얕은 멈춤도 경계로 잡게 할 수 있다.
