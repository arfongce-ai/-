# v3.31 분석 중 오류 수정 · 속도 안정화 교차검증

## 보고된 문제

1. 동영상 분석 시간이 오래 걸림.
2. 동영상 분석 중 오류가 발생함.

## 원인 분석

- **오류 원인:** MediaPipe `detectForVideo`는 호출 간 타임스탬프가 항상 증가해야 한다. 경계 탐색과 구간 분석에서 seek 오차로 같은 프레임을 두 번 검출하거나, 재분석으로 베이스 구간이 겹치면 비단조 타임스탬프가 들어가 분석이 중단되었다.
- **속도 원인:** v3.30에서 시험 도입한 순방향 재생 스캔(`forwardScanFrames`)이 `playbackRate = 1`로 동작해 영상을 실시간 길이만큼 재생해야 했고, 일부 WebView의 자동재생 제한·재생/일시정지 경쟁으로 오류 가능성도 있었다.

## 수정 사항

- 모든 포즈 검출을 `detectPoseAt` 래퍼로 통일했다. 제안 타임스탬프를 존중하되 직전 값 이하이면 +1ms 보정해 **항상 단조 증가**를 보장한다.
- 검출이 일시적으로 실패해도 빈 결과(`{ landmarks: [] }`)로 대체해 **분석 전체가 중단되지 않도록** 내성을 추가했다.
- 분석 시작 시 검출 타임스탬프 하한을 그 분석의 베이스 직전으로 초기화한다.
- 불안정한 순방향 재생 스캔을 제거하고, 검증된 빠른 seek 경로(`waitForSeek(t, true)`: seeked 직후 페인트 대기 없이 검출)로 되돌렸다.

## 정확성 보존

- 경계 밀도(초당 약 3.5회, 상한 320), 구간 표본 수(4~8), 재사용 허용오차(0.18초), 입력 해상도는 변경하지 않았다.
- 빠른 seek 경로는 좌표만 사용하고 표시 페인트가 불필요한 경계 탐색에 한정해 적용되며, 대표 장면 스냅샷은 기존 정밀 페인트 경로를 유지한다.
- 따라서 결과 프레임의 시각·좌표·개수와 리포트 점수는 검증된 안정 버전과 동일하다.

## 검증

- `node scripts/test-detect-timestamp.js` — 정상 증가, 같은/역행 입력, 베이스 경계 전환, 일시 검출 오류의 네 시나리오에서 단조 증가와 오류 내성을 확인했다.
- `npm run verify:cross` — 빠른 seek 스캔, `detectPoseAt` 래퍼, 타임스탬프 하한 초기화, 순방향 스캔 제거를 검증하며 통과한다.
- `assets:check`, `test:engine`, `test:mapping`(384회), `test:performance` 모두 통과한다.
- 페이지 로드 및 테마·모드 전환 중 콘솔 오류·경고가 없음을 확인했다.

## 참고

- 실시간 환경의 분석 시간은 영상 코덱·해상도·기기 성능에 따라 달라진다. 같은 영상을 다시 분석하면 안정화 캐시로 즉시 결과가 표시된다.
- 추가 속도가 필요하면 긴 영상은 품새 동작 구간만 잘라 입력하는 것이 가장 효과적이다.
