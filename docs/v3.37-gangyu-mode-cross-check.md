# v3.37 강유(剛柔) 분석 모드 교차검증

## 요청

- 강유 원리를 적용한 동작 최적화: (1) 앱에 강유 모드 구현, (2) 1페이지 체크시트 제작 — 둘 다.

## 1. 앱 — 강유 분석 모드

### 설계 원칙
강유는 추상 원리지만 영상에서는 **속도 곡선의 모양**으로 드러난다. 강(剛)은 짧고 가파른 정점, 유(柔)는 완만한 흐름. 앱이 이미 관절 속도·맺음·완급을 측정하므로, 그 위에 강유 판별을 얹는다. 기존 두 모드(수련/경기)를 건드리지 않고 **토글로 켜는 추가 분석**으로 구현해 위험을 최소화했다.

### 동작별 강유 판별 (`computeGangyu`)
구간의 프레임별 속도 배열에서:
- **가속 대비(accelRatio)** = 정점 속도 ÷ 준비 구간(앞 40%) 평균 속도. 완급의 핵심 지표.
- **정점 도달 프레임(framesToPeak)** = 동작 시작~정점까지 표본 수. 적을수록 날카로운 강.
- **분류**: accelRatio ≥ 2.2 & framesToPeak ≤ 3 → 강(剛) / accelRatio < 1.5 → 유(柔) / 그 사이 → 전환. 표본 부족 시 판정보류(안전).

각 구간 카드에 강/유/전환 + 가속 대비 + 정점 프레임을 표시한다.

### 강유 종합 + 리듬 체크리스트 (`gangyuSummary`)
리포트에 강유 카드를 추가해, 강/유/전환 분포와 완급조절 리듬 체크리스트 5가지를 영상 데이터로 채운다.
1. 정점 간격이 일정한가 — 강으로 분류된 정점들의 시작 간격 변동계수로 판정.
2. 준비는 느리고 끝은 빠른가 — 평균 가속 대비(2배 이상 목표).
3. 맺음마다 정지가 분명한가 — 맺음 평균 지표.
4. 연속/끊는 동작이 구분되는가 — 강·유·전환 분포(전환 많으면 정점 흐릿 신호).
5. 전체에 강약 물결이 보이는가 — 강·유 교대 여부.

### 적용 범위
토글은 수련품새·경기품새 **양쪽 모두**에서 동작하며, 모든 품새(18~30동작)에 적용된다. 분석 후에도 즉시 켜고 끌 수 있어 같은 결과를 강유 관점으로 다시 볼 수 있다.

## 2. 1페이지 체크시트 (별도 문서)

`강유 동작 최적화 체크시트`(Word, 1페이지)를 제작했다. 구성:
- 데이터 추출 가이드(관절 각도·임팩트 프레임·가속 대비·맺음 정지, 좋은 기준 포함)
- 강유 판별 로직(강·유 각각의 체크 질문)
- 완급조절 리듬 체크리스트 5가지
- 체계적 시작 5단계

앱의 강유 모드와 같은 기준을 쓰도록 맞췄다.

## 검증

- `node scripts/test-gangyu.js` — 가파른 정점→강, 완만→유, 중간→전환, 표본 부족→판정보류, 지표 유효 범위를 검증한다.
- 합성 속도 곡선 단위 검증: 강(가속 7.3배/3프레임), 유(1.13배), 전환(1.58배)으로 정확히 분류됨을 확인했다.
- `npm run verify:cross` — 강유 판별·종합 함수, 토글, 리듬 체크리스트 5항목을 검증하며 통과한다.
- 나머지 전체 테스트(assets/engine/mapping 384회/performance/timestamp/boundary/hevc) 통과.
- 페이지 로드·강유 토글 동작에서 콘솔 오류 없음.
- 체크시트 docx는 validate.py 통과, 1페이지 렌더 확인.

## 참고

- 강유 지표는 속도·맺음 기반의 코칭 지표이며, 실제 동작의 정오와 대회 감점은 지도자·심판 확인이 필요하다.
- 가장 정확한 강유 판별을 위해서는 정면·전신 촬영, 각 동작 끝에서 분명한 정지, H.264(MP4) 형식을 권장한다.
