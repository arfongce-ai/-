# v3.32 동작 경계 탐지 정확도 개선 교차검증

## 보고된 문제

- 부분별 영상에서 동작이 도중에 끊기거나 다음 동작으로 넘어감.
- 한 구간에 두 동작이 들어감.
- 동작명과 영상 속 동작이 다름.

## 원인

기존 경계 탐지는 영상을 동작 수로 **시간 균등 분할**한 뒤, 각 분할점 근처에서 정지점을 찾았다. 또 각 경계를 균등 간격의 ±38% 안으로 강제했다. 실제 품새는 동작마다 길이가 크게 다르므로(1초~5초), 이 균등 가정과 ±38% 제약이 경계를 실제 정지점에 도달하지 못하게 밀어냈다. 경계가 한 칸 밀리면 이후 동작명이 전부 한 칸씩 어긋났다.

## 수정 — valley prominence 기반 적응형 탐지

- 움직임의 **국소 최저점(valley)** 을 모두 찾고, 각 valley의 **prominence(앞뒤 봉우리 대비 깊이)** 를 계산한다. 동작 사이의 깊은 멈춤일수록 prominence가 크다.
- 깊은 valley부터 동작수−1개를 경계로 선택한다. **균등 간격 강제(±38%)를 제거**했다.
- prominence 임계를 **가장 깊은 valley 대비 비율로 자동 설정**(영상마다 적응)하고, 구간 감지 민감도 슬라이더로 미세 조정한다.
- 뚜렷한 멈춤이 동작 수보다 부족하면 **부족한 경계만 시간 균등으로 안전하게 메운다**(단조 증가·개수 보장, 절대 실패하지 않음).

## 사용자 안내 추가

- 균등 추정으로 메운 경계 비율이 1/3 이상이면 리포트 상단에 "동작 사이 멈춤이 뚜렷하지 않아 일부 경계를 시간 균등으로 추정했고, 동작명이 한 칸씩 어긋날 수 있다"는 안내를 표시한다.
- 각 구간의 경계 신뢰도(valley/uniform_fill, 상대 깊이)를 결과 데이터(`frameSamples`, `analysis_efficiency`)에 기록한다.

## 정확도 검증

합성 데이터(스캔 3.5/초)로 기존 균등분할 알고리즘과 200회 비교했다. 경계 평균오차(초):

| 시나리오 | 기존 평균 | 신규 평균 |
|---|---|---|
| 균등 | 0.15 | 0.15 |
| 앞 길고 뒤 짧음 | 0.33 | 0.15 |
| 중간에 긴 동작 | 0.13 | 0.09 |
| 극단적 불균등 | 0.33 | 0.18 |

- 잘 되던 균등 케이스는 동일하게 유지하고, 불균등 케이스의 평균오차를 약 절반으로 줄였다.
- 남은 최악값(약 0.7~0.8초)은 스캔 밀도(0.29초 간격)의 물리적 한계로, 기존 알고리즘도 동일하게 가진 한계다.

## 자동 검증

- `node scripts/test-boundary-detection.js` — 불균등 동작 평균오차 임계, 멈춤이 거의 없는 영상의 경계 개수·단조 증가 보장, 신뢰도 기록을 검증한다(index.html의 실제 함수를 추출해 실행).
- `npm run verify:cross` — valley prominence 적용, 신뢰도 기록, 균등 폴백, 동작명 밀림 안내, ±38% 제약 제거를 검증하며 통과한다.
- `assets:check`, `test:engine`, `test:mapping`(384회), `test:performance`, `test:timestamp` 모두 통과한다.
- 페이지 로드·모드 전환 중 콘솔 오류 없음.

## 참고

- 가장 정확한 결과를 얻으려면 각 동작 끝에서 잠깐(0.3~0.5초) 멈추고, 전신이 화면에 들어오게 촬영하는 것이 좋다. 멈춤이 뚜렷할수록 valley가 깊어져 경계가 정확해진다.
- 경계가 여전히 어긋나면 구간 감지 민감도를 높여 더 얕은 멈춤도 경계로 잡게 할 수 있다.
