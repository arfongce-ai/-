# v3.34 하이브리드 구간 탐지 교차검증 (전 품새 적용)

## 요청

- 다른 품새에도 적용 가능한지.
- 앞서 논의한 개선 A, B의 장점을 더하고 단점을 최소화하는 하이브리드로 정확도·신뢰성 향상.
- 그에 맞춰 설명도 수정.

## 전 품새 적용

경계 탐지·동작 배분 로직은 특정 품새에 묶이지 않고 `movements.length`(품새별 동작 수)를 기준으로 동작한다. 따라서 다음 17개 품새 모두에 동일하게 적용된다.

| 분류 | 품새(동작 수) |
|---|---|
| 유급자 | 태극 1~5장(18·18·20·20·20), 6장(19), 7장(25), 8장(27) |
| 유품자 | 고려(30) |
| 유단자 | 금강(27)·태백(26)·평원(21)·십진(28)·지태(28)·천권(26)·한수(27)·일여(23) |

## 하이브리드 구성 (A + B + 묶음)

세 기법을 결합하되, 각 기법이 서로의 단점을 막도록 순서를 둔다.

1. **개선 A — 준비자세 제외**: 활성 구간 시작을 "지속적 움직임이 시작되는 지점"으로 더 엄격히 잡아, 앞부분의 긴 준비 정지를 1번 구간에서 뺀다. 과도하게 잘라 첫 동작을 삼키지 않도록 전체의 25% 이내에서만 이동한다.
   - 단점(영상마다 효과 편차)은 B·묶음이 뒤에서 보완.
2. **강한 valley 자연 경계**: 깊은 멈춤만으로 끊는다(동작 수로 강제하지 않음).
3. **개선 B — 긴 구간 분할**: 평균의 1.7배를 넘는 긴 구간은, 내부에 실제 국소 최저점(임계 미달이라도)이 있을 때만 그 지점으로 추가 분할한다.
   - 단점(가짜 경계 생성)은 "실제 국소 최저점이 있을 때만 분할"로 차단.
4. **동작 묶음**: B로도 못 나누는 진짜 연속 동작(내부에 valley가 전혀 없음)은 한 구간에 묶어 동작명을 함께 표시한다.
   - 단점(정보 손실)은 A·B가 먼저 최대한 분리해 묶음을 최소화.

## 실제 영상 검증 (태극 2장)

업로드 영상에 하이브리드를 적용한 결과:
- 강한 valley만으로는 11개 구간 → **B 분할로 16개 구간**까지 분리.
- 진짜 연속 동작 **2개 구간만 묶음**으로 남음(이전엔 긴 구간 5개가 문제였음).
- 18개 동작 중 16개가 개별 표시, 2개만 묶여 표시.

## 단점 최소화 요약

| 기법 | 단점 | 막는 방법 |
|---|---|---|
| A | 영상마다 효과 편차 | B·묶음이 이후 단계에서 보완 |
| B | 가짜 경계 생성 위험 | 실제 국소 최저점이 있을 때만 분할 |
| 묶음 | 동작별 정보 손실 | A·B가 먼저 최대한 분리 |

## 설명 수정

- 구간 설명을 "N개 구간으로 나누어 평가"에서 "N개 동작을 멈춤 기준으로 나누고, 이어지는 동작은 묶어 동작명을 함께 표시. 각 동작 끝에서 잠깐 멈춰 촬영하면 더 정확히 나뉜다"로 바꿨다.
- 묶인 구간에는 "N~M번 동작 묶음" 태그와, 리포트 상단 안내를 표시한다.

## 자동 검증

- `node scripts/test-boundary-detection.js` — 자연 구간 범위·단조 증가, 동작 배분 정합성, 길이비례, **하이브리드 B 분할 동작**, **valley 없는 연속구간의 가짜 분할 금지**를 검증한다.
- `npm run verify:cross` — 개선 A·B 적용, 잔여 valley 활용, 동작 묶음 표시·안내를 검증하며 통과한다.
- `assets:check`, `test:engine`, `test:mapping`(384회), `test:performance`, `test:timestamp` 모두 통과한다.
- 페이지 로드 및 유급/유품/유단 전 품새 전환에서 구간 설명 정상 렌더링·콘솔 오류 없음.

## 촬영 팁

- 각 동작 끝에서 잠깐(0.3~0.5초) 분명히 멈추면 동작이 하나씩 분리된다.
- 시작 전 준비자세는 자동으로 제외되지만, 준비를 짧게 하면 더 안정적이다.
- 묶임이 많으면 구간 감지 민감도를 높여 더 얕은 멈춤도 경계로 잡게 할 수 있다.
- 영상이 HEVC(H.265)면 일부 기기에서 분석이 시작되지 않을 수 있으니 H.264 MP4 권장.
