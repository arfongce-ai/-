## v3.18 규정 피드백 및 구간 감지 개선

- 경기품새 피드백을 2026 경기규칙의 정확성 4.0점과 표현성 6.0점 기준에 연결했습니다.

- 분석 전에 영상 전체의 관절 움직임을 가볍게 탐색하고, 예상 경계 주변의 정지점으로 구간 경계를 보정합니다.
- 구간 감지 민감도를 `보통`, `높음(권장)`, `매우 높음`으로 선택할 수 있습니다.

## v3.17 확정 오류 수정

- 메인 영상과 팝업 구간 영상이 하나의 재생 범위를 공유하던 상태 충돌을 분리했습니다.
- 팝업 구간 재생을 시작할 때 메인 영상을 정지하여 영상과 소리가 중복 재생되지 않도록 했습니다.
- 실제 분석은 구간 길이에 따라 4~8개 표본을 사용하므로, 결과 데이터의 고정 `8개` 기록을 적응형 범위 기록으로 수정했습니다.
- 페이지를 닫거나 백그라운드에서 제거할 때 Blob URL을 해제하도록 정리 로직을 추가했습니다.
- 파일명과 선택 품새가 다르면 안내하되, 안내 확인 후 같은 분석 버튼 동작이 계속 진행되도록 수정했습니다.

## 수정한 문제

1. 정지 안정성 오판
   - 이전 방식은 속도의 표준편차만 사용하여 일정한 속도로 계속 움직여도 높은 정지 점수가 나올 수 있었습니다.
   - v3.17은 마지막 구간의 평균 잔여 속도와 표준편차를 함께 반영합니다.

2. 저사양 기기 분석 부하
   - 구간별 표본 수를 6~14개에서 4~8개로 낮추고, 일반 영상에서 초당 약 1.5~2회 이하가 되도록 제한했습니다.

3. 낮은 신뢰도의 신체 좌표
   - 어깨와 골반 좌표는 visibility 기준을 통과한 경우에만 신체 크기 정규화에 사용합니다.

4. 검증 문서와 실제 결과 불일치
   - 현재 헤더 문구와 검증 스크립트를 일치시켰습니다.
   - 실제 존재하는 자동 테스트 명령만 문서에 기록했습니다.

## 자동 검사

```text
npm run assets:check
npm run verify:cross
npm run test:engine
npm run test:all
```

`test:engine`은 다음을 검사합니다.

- 완전히 멈춘 장면은 높은 정지 점수를 받는지
- 일정한 속도로 계속 움직이는 장면이 높은 정지 점수를 받지 않는지
- 60초 및 90초 영상의 분석 표본 수가 초당 2회를 넘지 않는지
- 짧은 구간에도 동작 비교가 가능한 최소 표본이 유지되는지

## 실기기에서 반드시 확인할 항목

- 같은 영상과 같은 품새를 반복 분석했을 때 결과가 동일한지
- 구간 카드를 누르면 해당 장면이 재생되는지
- 다른 파일 또는 품새 선택 후 새 분석을 정상적으로 시작할 수 있는지
- 저사양 Android에서 분석 중 앱 종료나 과도한 발열이 없는지
- iOS Safari에서 화면 전환 후 영상과 분석 상태 안내가 올바른지
