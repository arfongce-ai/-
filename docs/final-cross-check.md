# v3.16 최종 교차검증

## 결론

분석 신뢰도 보완에 더해 모바일 구간 확인 흐름과 훈련 분석 요약 저장 기능을 개선했습니다.

## v3.16 UI/UX 보완

- 파일명에 표시된 품새와 선택한 품새가 다르면 경고합니다.
- 재분석(초기화) 버튼을 화면에서 숨겼습니다.
- 구간 카드를 누르면 화면 이동 없이 팝업에서 영상을 재생합니다.
- `결과지`를 `훈련 분석 요약`으로 변경했습니다.
- 훈련 분석 요약 상단에서 A4 요약 JPG를 만들 수 있습니다.

## 수정한 문제

1. 정지 안정성 오판
   - 이전 방식은 속도의 표준편차만 사용하여 일정한 속도로 계속 움직여도 높은 정지 점수가 나올 수 있었습니다.
   - v3.16은 마지막 구간의 평균 잔여 속도와 표준편차를 함께 반영합니다.

2. 저사양 기기 분석 부하
   - 구간별 표본 수를 6~14개에서 4~8개로 낮추고, 일반 영상에서 초당 약 1.5~2회 이하가 되도록 제한했습니다.

3. 낮은 신뢰도의 신체 좌표
   - 어깨와 골반 좌표는 visibility 기준을 통과한 경우에만 신체 크기 정규화에 사용합니다.

4. 검증 문서와 실제 결과 불일치
   - 현재 헤더 문구와 검증 스크립트를 일치시켰습니다.
   - 실제 존재하는 자동 테스트 명령만 문서에 기록했습니다.

## 자동 검사

```text
npm run assets:check
npm run verify:cross
npm run test:engine
npm run test:all
```

`test:engine`은 다음을 검사합니다.

- 완전히 멈춘 장면은 높은 정지 점수를 받는지
- 일정한 속도로 계속 움직이는 장면이 높은 정지 점수를 받지 않는지
- 60초 및 90초 영상의 분석 표본 수가 초당 2회를 넘지 않는지
- 짧은 구간에도 동작 비교가 가능한 최소 표본이 유지되는지

## 실기기에서 반드시 확인할 항목

- 같은 영상과 같은 품새를 반복 분석했을 때 결과가 동일한지
- 구간 카드를 누르면 해당 장면이 재생되는지
- 다른 파일 또는 품새 선택 후 새 분석을 정상적으로 시작할 수 있는지
- 저사양 Android에서 분석 중 앱 종료나 과도한 발열이 없는지
- iOS Safari에서 화면 전환 후 영상과 분석 상태 안내가 올바른지
