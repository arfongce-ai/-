# v3.35 리포트 모드별 정비 (묶음 구간 반영) 교차검증

## 요청

- 구간이 "동작 묶음"으로 바뀌었으니 리포트도 함께 수정.
- 수련품새: 동작의 확실성 + 문제점을 초등학생도 이해하기 쉽게.
- 경기품새: 동작의 가감점이 명확 + 수정·보완의 핵심 포인트.

## 수련품새(exam) 변경

- **동작 확실성 표시**: 각 구간에 "동작이 잘 보이나요?"를 추가하고, 감지율과 묶음 여부로 네 단계(또렷함 / 대체로 또렷 / 이어진 동작 / 또렷하지 않음)로 쉽게 알려준다. 각 단계마다 쉬운 말 설명을 함께 제공한다.
- **쉬운 말 문제·수정**: 기존의 "무엇이 아쉬운지 / 이렇게 고쳐요"를 유지·강화한다.
- **묶음 안내**: 두 동작이 한 구간에 들어가면 "두 동작이 이어졌어요. 동작 끝에서 잠깐 멈추면 하나씩 또렷해져요"라고 알린다.
- **요약**: 또렷하게 잘 된 동작 수 / 전체, 이어진 동작 수를 표시한다.

## 경기품새(competition) 변경

- **가감점 명확화**: 각 구간에 "가감점: 약 -0.1/-0.3점 가능성" 또는 "판정 보류(촬영/지도자 확인)"를 명시하고 사유를 함께 표시한다.
- **핵심 수정 포인트**: 점수를 가장 많이 되돌릴 한 가지(가장 약한 측정 지표: 완급/맺음/마무리 안정/서기 정렬)를 골라 "핵심 수정"으로 짚는다.
- **가감점 합계(추정)**: 요약에 큰 오류 -0.3 곳수, 경미한 오류 -0.1 곳수, 판정 보류 곳수와 합계 추정치를 표시한다. 실제 감점은 심판이 확정한다고 명시한다.
- **묶음 안내**: 동작이 이어진 구간은 "감점은 동작별로 나눠 지도자 확인 권장"이라고 안내한다.

## 출력물 동기화

- 화면 카드, 요약, 그리고 텍스트(JSON/복사용) 리포트 모두 동작 확실성·가감점·핵심 수정·묶음 표시를 반영하도록 수정했다.

## 검증

- 핵심 헬퍼(`movementCertainty`, `keyFixPoint`)를 추출해 단위 검증: 또렷/이어진 동작/흐릿 판정과, 가장 약한 지표 기반 핵심 수정 선택이 정확함을 확인했다.
- `npm run verify:cross` — 동작 확실성 함수·표시, 핵심 수정 함수·표시, 가감점 명시, 가감점 합계, 확실성 요약, 묶음 안내를 검증하며 통과한다(256개 점검).
- `assets:check`, `test:engine`, `test:mapping`(384회), `test:performance`, `test:timestamp`, `test:boundary` 모두 통과한다.
- 페이지 로드·모드/품새 전환에서 콘솔 오류 없음.

## 참고

- 동작 확실성은 감지율과 묶음 여부 기반의 코칭 지표이며, 실제 동작의 정오는 지도자 확인이 필요하다.
- 가감점은 영상 좌표로 측정 가능한 지표 기반의 참고값이며, 실제 감점은 심판이 확정한다.
