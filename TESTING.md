# AI 품새 코치 테스트 가이드

## 테스트 APK

테스트용 APK 사본:

```text
AI-poomsae-coach-test.apk
```

원본 debug APK:

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

## 설치 전 확인

- 기존에 설치된 `AI 품새 코치`가 있으면 먼저 삭제합니다.
- 휴대폰 설정에서 `알 수 없는 앱 설치 허용`을 켭니다.
- 카카오톡 전송이 막히면 USB 케이블 또는 Google Drive로 옮깁니다.
- Play 프로텍트 경고가 뜨면 테스트용 debug APK라서 발생하는 정상 경고입니다.

## 빠른 기능 테스트

1. 앱 실행
2. 품새 선택
3. 영상 파일 선택
4. 모델 준비
5. 평가 시작
6. 구간별 리플레이 버튼 클릭
7. 왼쪽 영상이 바로 재생되는지 확인
8. 속도 0.5x, 0.75x, 1.0x 확인
9. 결과지 보기 버튼 클릭
10. 결과지 별도 화면 확인
11. 훈련 화면으로 버튼 클릭
12. JPG 결과지 생성 확인

## 이번 UI/UX 변경 확인

- 영상 영역이 왼쪽에 유지되는지 확인합니다.
- 구간별 평가 목록을 스크롤해도 영상 확인이 쉬운지 확인합니다.
- 리포트 출력 textarea가 메인 화면에 보이지 않는지 확인합니다.
- 결과지는 별도 페이지로 열리는지 확인합니다.
- JPG 결과지 생성 후 미리보기가 결과지 화면에 나오는지 확인합니다.

## Android 실기기 체크

- 카메라/파일 권한 요청이 정상인지 확인합니다.
- 앱 실행 후 WebView 화면이 깨지지 않는지 확인합니다.
- 90초 이하 영상에서 분석이 끝까지 진행되는지 확인합니다.
- 모델 준비 실패 시 오류 문구가 표시되는지 확인합니다.
- GPU 실패 기기에서 CPU 폴백으로 모델 준비가 재시도되는지 확인합니다.

## 문제가 생길 때

### 앱 설치가 안 됨

- 기존 앱 삭제 후 다시 설치합니다.
- APK를 카카오톡이 아닌 USB 또는 Google Drive로 옮깁니다.
- Play 프로텍트에서 `세부정보 더보기` 후 `무시하고 설치`를 선택합니다.

### 분석 중 멈춤

- 같은 시간 seek 데드락 방지 패치가 들어가 있습니다.
- 그래도 멈추면 영상 길이와 파일 형식을 바꿔 테스트합니다.
- MP4 영상, 30초 내외 영상으로 먼저 확인합니다.

### 모델 준비 실패

- 오프라인 자산이 포함되어 있는지 확인합니다.
- `npm.cmd run assets:check`를 실행해 자산 상태를 확인합니다.

## 개발자 검증 명령

```powershell
npm.cmd run assets:check
npm.cmd run android:sync
cd android
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
.\gradlew.bat assembleDebug
```
